# Web-App for HR (Alabuga Case)

Веб-приложение для автоматизации подбора персонала. Реализует взаимодействие кандидатов, HR-специалистов и руководителей в формате «маркетплейса», где резюме выступает карточкой товара.

## Функционал

**Кандидат**
- Регистрация и вход
- Заполнение резюме (ФИО, контакты, должность, зарплатные ожидания, опыт, образование, навыки, компетенции, языки, проекты, сертификаты, график работы)
- Загрузка и удаление документов (PDF, DOCX, PNG, JPG)
- Просмотр своего статуса
- Получение уведомлений о приглашениях на собеседование, офферах и отзывах
- Принятие или отклонение приглашений

**HR**
- Просмотр всех резюме с фильтрацией по должности, статусу, зарплате и формату работы
- Добавление резюме в корзину
- Назначение собеседований с выбором руководителя, даты, формата и места/ссылки
- Отправка произвольных уведомлений кандидатам
- Получение уведомлений о запросах собеседований от руководителей и решениях кандидатов
- Отметка уведомлений прочитанными

**Руководитель**
- Просмотр резюме
- Запрос на проведение собеседования (уведомление HR)
- Оставление отзыва с оценкой по 10‑балльной шкале и рекомендацией
- Отметка о прохождении собеседования, отправка оффера, приём кандидата
- Получение уведомлений о назначении на собеседование и ответах кандидатов

## Установка и запуск (с нуля)

### 1. Установка Python 3.12

1. Перейдите на [python.org/downloads](https://www.python.org/downloads/)
2. Скачайте установщик для macOS (Python 3.12.x)
3. Запустите скачанный `.pkg`-файл и следуйте инструкциям установщика
4. После установки откройте **Терминал** и проверьте версию:

```bash
python3.12 --version
```

### 2. Установка Node.js

Перейдите на nodejs.org
Скачайте LTS-версию для macOS (рекомендуется 20.x или выше)
Запустите .pkg-файл и выполните установку
Проверьте установку в Терминале:
bash
node --version
npm --version
### 3. Установка PostgreSQL 17

Скачайте установщик для macOS с postgresql.org
Выберите версию 17 и запустите .dmg-файл
В процессе установки запомните пароль, который вы зададите для суперпользователя postgres
После завершения установки PostgreSQL будет автоматически запущен как служба
Добавьте путь к командам PostgreSQL в системный PATH. Откройте Терминал и выполните:
```
bash
echo 'export PATH="/Library/PostgreSQL/17/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
```
Проверьте доступность команды psql:
```
bash
psql --version
```
Если команда не найдена, перезапустите Терминал.

### 4. Клонирование репозитория

Скачайте архив проекта или выполните в Терминале:
```
bash
git clone <URL_репозитория> project
cd project
```
### 5. Настройка и запуск бэкенда

#### 5.1. Создание виртуального окружения Python
```
bash
python3.12 -m venv venv
source venv/bin/activate
```
#### 5.2. Установка зависимостей Python
```
bash
cd backend
pip install -r requirements.txt
```
Содержимое requirements.txt (должно находиться в папке backend):
```
text
fastapi==0.115.6
uvicorn[standard]==0.34.0
sqlalchemy==2.0.36
psycopg2-binary==2.9.10
python-jose[cryptography]==3.3.0
bcrypt==4.2.0
python-multipart==0.0.20
pydantic-settings==2.6.1
python-dotenv==1.0.1
aiofiles==24.1.0
email-validator==2.2.0
```
#### 5.3. Создание базы данных и пользователя

Подключитесь к PostgreSQL под суперпользователем postgres:
```
bash
psql -U postgres
```
Введите пароль, заданный при установке PostgreSQL.

Внутри psql выполните команды:
```
sql
CREATE USER hr_user WITH PASSWORD 'hr_pass';
CREATE DATABASE hr_db OWNER hr_user;
GRANT ALL PRIVILEGES ON DATABASE hr_db TO hr_user;
\q
```
#### 5.4. Файл конфигурации .env

В папке backend создайте файл .env со следующим содержимым:
```
text
DATABASE_URL=postgresql://hr_user:hr_pass@localhost/hr_db
SECRET_KEY=любая-длинная-случайная-строка
ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=30
UPLOAD_DIR=./uploads
```
Замените любая-длинная-случайная-строка на свой секретный ключ.

#### 5.5. Запуск бэкенда

Убедитесь, что виртуальное окружение активировано (приглашение (venv)), и выполните:
```
bash
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
Сервер запустится по адресу http://localhost:8000.
Интерактивная документация API: http://localhost:8000/docs.
```
### 6. Настройка и запуск фронтенда

Откройте второе окно Терминала и перейдите в папку frontend:
```
bash
cd project/frontend
```

#### 6.1. Установка зависимостей Node.js
```
bash
npm install
```
#### 6.2. Запуск фронтенда
```
bash
npm run dev
Приложение откроется по адресу http://localhost:5173.
```

### 7. Первый вход

Перейдите на http://localhost:5173/register
Зарегистрируйте пользователя с ролью Candidate (кандидат), HR или Manager (руководитель)
Войдите и начните использование
